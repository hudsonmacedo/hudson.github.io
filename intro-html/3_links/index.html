Hyperlinks

↑
↑

A linguagem tem "Hypertext" no nome, então essa é uma parte importante de saber: como conectar páginas, arquivos, scripts, imagens, etc...

Existem alguns tipos diferentes de links que podem ser colocados dentro de um HTML, cada um com sua função.

↑
↑

Para outra página

↑
↑

O tipo mais básico de link são aqueles inseridos em menus e no meio dos textos, que direcionam para outra página. Para criar esse link, é necessário usar o atributo href na tag <a>. Esse atributo também vale para outras tags, como <link>

<a href="index.html"> index </a> <link href="style.css">
↑
↑

Relativo vs Absoluto

↑
↑

O valor do atributo href pode ser indicado de 2 maneiras: link relativo ou absoluto.


Relativo: Refere-se a um outro arquivo que está dentro do mesmo domínio. Se o arquivo mudar de pasta, ele perde o link.

<a href="index.html"> index </a>

Absoluto: Refere-se a um arquivo que pode estar em qualquer lugar da web.

<a href="http://movile.com/index.html"> index </a>
↑
↑

E-mail

↑
↑

Alguns sites usam um tipo especial de link para direcionar o usuário para eu cliente de e-mail, sem que ele precise copiar e colar o endereço, colocando o valor mailto.

<a href="mailto:matiazzo@movile.com"> E-mail </a>

Problema: Essa função direciona para clientes de e-mail, ou seja, em um computador vai abrir o seu Outlook (windows) ou Mail (mac) e esses programas nem sempre estão configurados para uso.

↑
↑

Nova aba

↑
↑

Provavelmente já viram links que carregam na mesma aba que você está navegando e outros que abrem uma aba nova com o endereço. Por padrão, os links abrem na mesma aba, mas isso pode ser alterado, indicando o valor _blank para o atributo target para o browser.

<a href="index.html" target="_blank"> E-mail </a>
↑
↑

âncora: link na mesma página

↑
↑

Os links que direcionam para um seção da mesma página são chamados de âncora. São muito utilizados em LP de página única para scrollar até o item desejado. Para fazer esse link basta por o #id do elemento referência no valor.

<a href="#features""> Features </a>
<section id="features""> ... </section>

Com isso, o browser vai alterar a altura da página até que a section features esteja no topo.

  Esse é um dos motivos pra não repetir um ID na página.
