Elementos inline

↑
↑

Elementos inline ocupam, na página, o espaço de seus conteúdos. Ao contrário dos block, que ocupam a largura toda, os inline vão se alinhando um ao lado do outro. Eles podem estar tanto um dentro do outro quanto dentro de um elemento block.

É comum usarmos elementos inline para dar destaque a algumas palavras, com sublinhados, negritos, itálicos ou com cores diferentes.


Elemento inline que tem um elemento inline dentro dele e os dois estão dentro de um elemento block que agrupa os dois.

Semântica

↑
↑

Semântica, no HTML, é a prática de dar significado aos elementos da página. Dos benefícios existentes ao trabalhar seu código de modo semântico, os principais são: possibilitar que devices leiam seu código para, por exemplo, mostrarem a prévia de um texto num feed ou permitir que softwares de acessibilidade leiam, literalmente, sua página para deficientes visuais; e facilitar a leitura do próprio código por outros desenvolvedores (evitando o retrabalho).

<div> e <span>

↑
↑

div e span são dois elementos essenciais para qualquer código HTML. Mas eles não tem nenhum valor semântico e servem como containers para outros elementos.

Por exemplo, parágrafos são colocado dentro da tag <p> e títulos são colocados nas tags <h1> até <h6> por serem headings.

div é um elemento block e span, inline.

Classes e ids

↑
↑

É normal ver divs e spans com atributos de classe e id. Esses atributos servem pra "nomear" esses elementos para que eles possam ser alvos de links e estilos.


Classes: A mesma classe pode ser aplicada em mais de um elemento.

<a class="menu"> Movile </a>
<span class="menu"> Movile </span>

Tanto a tag <a> quanto a <span> tem a classe menu


Ids: também podem ser usados para estilizar, mas apenas 1 elemento.

<div id="intro"> Introdução da página </div>
<div id="about"> Sobre o site </div>
<div id="intro"> Outro conteúdo </div>

Posso repetir um id? Clique aqui pra saber. IDs são como serial number de produtos, são usados para identificar um elemento único na página.

↑
↑

nomeando classes e ids

↑
↑

Uma das metodologias mais disseminadas diz que as classes e os ids devem ser nomeados em relação à função que os elementos cumprem, e não à sua aparência. Ou seja, se formos criar uma div com fundo azul para as redes sociais, escreveríamos:

<div class="social"> facebook | twitter </div>

e não:

<div class="fundo-azul"> facebook | twitter </div>

Assim, se essa div mudar de fundo azul pra vermelho no futuro, o nome "fundo-azul" não fará mais sentido mas o "social" sim.

Estruturando

↑
↑

Por muito tempo o único jeito de contruir páginas era com <div> e isso ainda é possível, porém, é semânticamente prejudicial.

Com a atualização do HTML 5 foram criados alguns novos elementos block, que têm a mesma função de uma div, mas são mais semânticos.


Header: usado para topos de páginas ou seções.

Nav: identifica navegações principais (menu).

Article: conteúdo independente, que pode ser usado fora da página. Post de blog, notícia, conteúdos em geral.

Section: agrupa conteúdos com mesmo tema

Aside: para sidebars ou curtas explicações (autor do texto)

Footer: autoexplicativo
